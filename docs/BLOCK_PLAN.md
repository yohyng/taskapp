# Task Space — ブロック構成 計画表

## 🎯 ゴール
Notionの「ブロック」という考え方を **土台** にしつつ、タスク管理に特化して **自分用にリッチにカスタマイズ** した、列レイアウトの個人タスクOS。
Notionそのものの再現ではなく、Notionの **操作感**（⋮⋮ハンドル・ネスト・挿入ライン・スラッシュ追加）を借りて、今のアプリ独自の強み（複数表示・繰り返しゴースト・列ビュー）を伸ばす。

## ✅ 決定済みの方針（矛盾の解消）
| 論点 | 決定 |
|---|---|
| データモデル | **見た目だけNotion化**。1タスク複数表示は維持。データモデルは現状のまま |
| 繰り返しの7Days表示 | **参照（ゴースト）として残す**。実体ブロックはプロジェクト側に1つ |
| レイアウト | **列レイアウトを維持**（5列 / ボード / 7Days）。縦1カラム化はしない |

---

## 🧱 「ブロック」の定義（このアプリ版）
すべてのタスク表示を共通の **Block** コンポーネントに統一する。Blockは以下を持つ：

- **種別 (type)**: `todo`（既定）/ `heading` / `text` / `divider` / `callout`
- **左端の⋮⋮ハンドル**（ホバーで出現、ドラッグ起点）
- **チェックボックス**（todoのみ）
- **本文**（タイトル）
- **メタチップ**: プロジェクト名・期日・繰り返し・メモアイコン
- **ネスト**（子ブロックをインデント表示）
- **ゴーストフラグ**: 参照表示（繰り返し由来）は薄く＋リンクアイコン

---

## 🗺 フェーズ計画

### Phase 1 — 操作感のNotion化（最優先・データ変更なし）
1. **DragOverlay幅崩れの修正**（現状の縦長バグ） … 明示的な幅・block表示
2. **共通 `Block` コンポーネント** 抽出（Today/Weekly/Project/7Days/DayTaskを統一）
3. **⋮⋮ ハンドル** をホバー時に左端表示。ハンドルからのみドラッグ開始（本文クリックは選択/編集）
4. **青い挿入ライン** … ドロップ位置（上/下/子）を明示するインジケータ
5. **インデントでネスト** … ドラッグの右寄せ or Tab/Shift+Tab で親子変更

### Phase 2 — リッチ化（ブロック種別と操作メニュー）
6. **⋮⋮ クリックでブロックメニュー** … 種別変更 / 色 / 複製 / 削除 / 〜へ移動
7. **スラッシュメニュー (`/`)** … 見出し・テキスト・区切り線・callout を挿入
8. **見出し / テキスト / 区切り** ブロック … プロジェクト内をセクション分け
9. **ブロックごとの色・絵文字**（プロジェクトの仕組みを流用）

### Phase 3 — 参照（ゴースト）の整備
10. **繰り返しゴースト** を「参照ブロック」として明示（薄色＋🔗）。完了操作は実体に反映
11. **ゴーストのドラッグ抑制**（実体はプロジェクト側で操作）

---

## ⚠️ 残課題・要確認（実装中に出たら都度相談）
- ネストの最大深さ（現状 `MAX_DEPTH`）をNotion風に深くするか維持か
- `heading`/`text` ブロックは「タスクではない」ため、件数カウントや完了集計から除外する扱い
- スラッシュメニューはPC前提（モバイルは ＋ ボタン併用）

---

## 進め方
Phase 1 から着手。まず **DragOverlayの修正 + 共通Block + ⋮⋮ハンドル + 挿入ライン** を1セットで実装してレビュー → OKなら Phase 2 へ。
